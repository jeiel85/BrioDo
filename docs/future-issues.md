# BrioDo 프로젝트 온전화 및 고도화를 위한 종합 백로그

현재 BrioDo는 코어 기능(AI 스마트 입력, 잠금화면, 동기화)과 기초 보안을 성공적으로 갖춘 훌륭한 상태입니다. 이 프로젝트를 **상업적으로도 성공하고, 기술적으로도 무결점인 프로덕트**로 만들기 위해 4가지 핵심 영역(보안, 수익, 마케팅, 사용자)에서 처리해야 할 과제들을 종합 이슈 형태로 정리했습니다.

---

## 🛡️ 1. 보안 및 인프라 안정성 (Security & Infrastructure)

> API 키를 백엔드(Cloud Functions)로 숨겼으나, 누군가 악의적으로 함수 엔드포인트를 무한 호출할 수 있는 취약점이 남아 있습니다.

- **[Issue-Sec-1] Firebase App Check (Play Integrity API) 도입**
  - **상세**: Cloud Functions(`generateGeminiContent`)가 오직 "Google Play에서 정식으로 다운로드 된 BrioDo 앱"에서만 호출되도록 인증 처리.
  - **기대 효과**: 외부에서의 API 무단 호출 완전 방어, Gemini API 계정의 과금 폭탄 사전 차단.

- **[Issue-Sec-2] AI 프롬프트 내 개인정보(PII) 비식별화 처리**
  - **상세**: 사용자가 "홍길동 계좌(110-123-4567)로 송금"이라고 음성을 남길 경우, AI 서버 전송 전에 기기 로컬에서 정규식으로 계좌/전화번호 등을 마스킹(`***`) 처리.
  - **기대 효과**: 사용자 프라이버시 보호 및 민감정보 전송 가이드라인 준수.

- **[Issue-Sec-3] Firestore Security Rules 엄격화 제어**
  - **상세**: 다른 유저의 데이터를 절대 읽고 쓰지 못하도록 방어하고, Firestore Write/Read에 대한 **분당 호출 제한(Rate Limiting)** 규칙 추가로 어뷰징 차단.

## 💰 2. 수익 모델 구축 (Monetization)

> 다소 복잡했던 초기 '브리오(Brio)' 재화 모델을 걷어낸 상태입니다. 완전 무료 제보는 초기 유입에 좋으나 장기 서비스를 위한 서버 유지 수익이 필수적입니다.

- **[Issue-Mon-1] Google Play 빌링 연동 (BrioDo Pro / 프리미엄 구독)**
  - **상세**: 일 10회 AI 제한을 무제한으로 해제하고, '프리미엄 전용 테마', '다기기 실시간 추가 기기' 등을 묶어 월 $1.99~2.99 수준의 저렴한 구독 모델 출시. (`@capacitor-community/in-app-purchases` 플러그인 활용)
  - **기대 효과**: 배너 광고 없는 깔끔한 경험을 유지하면서 안정적 서버 유지비 일괄 확보.

- **[Issue-Mon-2] 커피 한 잔 후원하기 (자발적 결제)**
  - **상세**: 기능 제한 없이 사용하려는 코어 유저들을 위해, 설정 화면 하단에 아기자기한 애니메이션과 함께 [개발자에게 커피 사주기] 1회성 인앱 결제 버튼 추가.

- **[Issue-Mon-3] 네이티브 광고(Native Ads) 리스트 삽입 (대안)**
  - **상세**: 사용성을 심하게 해치는 배너 대신, 할 일 목록 컴포넌트 사이에 UI/UX를 해치지 않는 Card 형태의 네이티브 중심 광고 간헐적 삽입.

## 🚀 3. 마케팅 및 오가닉 메트릭스 (Marketing & Growth)

> 투두 앱은 경쟁이 매우 치열한 레드오션입니다. "잠금화면"과 "AI 언어 인식"이라는 BrioDo만의 차별화된 강력한 무기를 전면에 어필해야 합니다.

- **[Issue-Mkt-1] 앱스토어 최적화(ASO) 및 타겟 키워드 점령**
  - **상세**: 스토어 앱 제목을 "BrioDo" 단일 단어에서 **"BrioDo - AI 빙의 잠금화면 할 일 관리"** 등으로 부연 설명을 붙여 변경.
  - **기대 효과**: 연관 키워드("잠금화면 할일", "AI 투두", "음성 다이어리")를 검색하는 오가닉(자연 검색) 다운로드 비율 300% 이상 상승.

- **[Issue-Mkt-2] "자랑하기(Share)" 소셜 바이럴 루프 구축**
  - **상세**: 100개 완료, 7일 연속 달성 등 '희귀 업적' 달성 시, 화려한 디자인 템플릿 이미지를 제공하여 인스타그램 스토리 / 트위터에 쉽게 공유할 수 있는 기능 도입. (`html2canvas`로 이미지 캡처 후 Capacitor Share API 연동)
  - **기대 효과**: 사용자들의 자발적인 SNS 인증을 통한 무료 바이럴 트래픽 확보.

- **[Issue-Mkt-3] 메이커 커뮤니티 런칭 전략 수립**
  - **상세**: 앱 안정화 후, 한국의 IT 제품 커뮤니티인 '디스콰이엇(Disquiet)'과 글로벌 '프로덕트 헌트(Product Hunt)'에 공식 포스팅 메이커 스토리 업로드.

## ❤️ 4. 사용자 경험 및 리텐션 강화 (UX & Retention)

> 기능은 월등하나 지속적으로 접속하게 만드는(Retention) '습관 형성' 및 기기 범용성에 대한 대안이 필요합니다.

- **[Issue-UX-1] 안드로이드 데스크톱(홈 화면) 위젯 개발 (최우선 과제)**
  - **상세**: 잠금화면 알림도 강력하지만, 안드로이드 사용자는 "홈 화면 바탕화면 위젯"을 매우 선호합니다. Android Native (Kotlin/Java) 코드로 `AppWidgetProvider` 브릿지를 개발하여 바탕화면 위젯 제공.

- **[Issue-UX-2] 잔디 심기(Streak) 및 통계 시각화**
  - **상세**: 깃허브(GitHub) 잔디 컴포넌트처럼 '매일 완료한 할 일 수'를 색상 강도로 렌더링. "현재 12일째 연속 달성 중!" 🔥 등의 문구를 제공.
  - **기대 효과**: 사용자가 목표 끈이 끊어지는 것을 막고자 매일 접속하며 이탈율(Churn Rate)을 획기적으로 낮춤.

- **[Issue-UX-3] PC 웹 브라우저 접속 (PWA 도메인 지원)**
  - **상세**: 직장인과 학생은 업무/수업 중에는 모바일보다 PC로 할 일을 관리합니다. Firebase Auth와 Firestore를 이미 활용하고 있으므로 URL(`app.briodo.com` 등) 호스팅만으로 데스크톱 레이아웃 PWA를 제공 가능함.

- **[Issue-UX-4] AI "묵은 할 일" 스마트 정리 제안**
  - **상세**: 생성된 지 2주가 지나도 미완료로 남겨진 일정이 확인될 경우, 아침 AI 브리핑 시 "이 일정을 오늘로 미룰까요, 아니면 과감히 삭제할까요?"라고 넛지(Nudge)해주는 기능 추가.
