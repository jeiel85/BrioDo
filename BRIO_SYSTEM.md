> 수익화 모델 + 브리오 크레딧 시스템 전체 설계 문서

---

## 1. 철학

| 원칙 | 내용 |
|------|------|
| 오픈소스 | 코드 전체 공개 유지 |
| 배너 광고 없음 | 앱 경험 최우선 |
| 구독/유료앱 없음 | 진입 허들 최소화 |

## 2. 브리오(⚡) 기본 규칙

```
브리오 = 앱 내 에너지 단위 (크레딧)
기본 제공: 매일 자정 10⚡ 자동 충전
최대 보유: 제한 없음 (잔량 누적 허용)
```

### 2-1. 소모처

| 기능 | 소모량 |
|------|--------|
| AI 자연어/음성 입력 1회 | ⚡ 1 |
| AI 태그 자동 추출 | ⚡ 1 |
| Google 캘린더 수동 동기화 | ⚡ 1 |

> 기본 할 일 CRUD, 수동 입력, 테마 변경 등은 무제한 무료

### 2-2. 충전 방법

| 방법 | 충전량 | 조건 |

| 업적 달성 보상 | ⚡ 1~50 | 난이도별 차등 |
| 프리미엄 기능 해제 광고 | (별도) | 7일 기능 활성화 |

## 3. 광고 단가 분석 (보상형 광고 1편당 브리오 ⚡5 근거)

```
한국 보상형 광고 eCPM: $8~$15 (평균 $10)
광고 1편당 수익: $10 ÷ 1000 = $0.01

Gemini 2.5 Flash Lite API 1회 원가:
  입력 ~500 tokens × $0.075/1M = $0.0000375
  출력 ~200 tokens × $0.30/1M  = $0.00006
  합계: ≈ $0.0001/회

광고 1편($0.01) ÷ API 원가($0.0001) = 100배 여유

⚡5 지급 기준:
  - ⚡1 = AI 1회 = $0.0001 원가
  - ⚡5 = $0.0005 원가 → 광고 수익($0.01) 대비 20배 마진
  - 캘린더 동기화 등 추가 비용 고려해도 충분한 수익성
```

**결론: 광고 1편 = ⚡5 충전 (적정 균형점)**

## 5. UI 배치 계획

### 5-1. 헤더 브리오 표시
```
[날짜 영역]    [⚡ 7]   [검색] [설정]
```
- 브리오 잔량 탭 → 충전 팝업 오픈

### 5-2. 브리오 부족 팝업 (충전 모달)
```
┌─────────────────────────────┐
│  ⚡ 브리오가 부족해요         │
│                              │
│  광고를 보고 ⚡5를 충전할까요? │
│                              │
│  [ 광고 보기 (+⚡5) ]        │
│  [ 오늘은 그냥 쓸게요 ]       │
└─────────────────────────────┘
```

### 5-3. 업적 달성 시 브리오 보상 알림
기존 업적 잠금해제 모달에 보상 브리오 표시 추가:
```
🎉 업적 달성!
━━━━━━━━━━━━━━
🏆 일주일 챔피언
⚡ +5 브리오 획득!
```

### 5-4. SmartInputModal 상단 브리오 표시
```
┌──────────────────────────────┐
│  ✨ AI 입력   ⚡ 7 / 10     │
│  (브리오 1개가 소모됩니다)     │
└──────────────────────────────┘
```

## 8. 구현 단계 (AdMob 없는 MVP 먼저)

AdMob SDK 연동 전에도 브리오 시스템은 동작해야 함:

```
Phase 1 (현재): 브리오 UI + 로직만 구현
  → 광고 버튼 클릭 시 "준비 중" 또는 테스트용 즉시 +5 지급

Phase 2 (AdMob 연동 후): 실제 광고 영상 시청 후 +5 지급
  → ADMOB_PLAN.md 절차 참조
```
